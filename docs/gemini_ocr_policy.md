# Gemini OCR 運用メモ

## ハルシネーション検知ルール（2026-04-18 追加）

**初回の Gemini OCR 実行で `branches = 0` が返り、リトライ実行で肢ありの結果が返った場合、リトライ結果は「捏造（ハルシネーション）疑い」として自動採用しない。** 同一画像に対する 0 肢→複数肢の揺れは、章扉・目次・白紙ページを誤ってキャプチャした際に 2 回目以降のリクエストで存在しない問題文を生成する既知の挙動と整合する。採用判断は必ず実画像の目視確認を前提とし、0 肢回答を正とみなすのが安全側の既定とする。

### 発生事例

- **p018（KB2025、2026-04-18）**: 1回目 branches=0（正）、503 エラー後のリトライで branches=5（全肢捏造、実画像は「II. 行政法」章扉）。候補から除外し `p018 = 取込不要` で確定。

### 運用手順

1. 0 肢が返ったページは、まず `~/Desktop/kindle_shots/{page}.png` を目視確認
2. 章扉・目次・白紙と確認できた場合は `取込不要` として記録、リトライしない
3. どうしても OCR が必要な場合は、画像トリミングで問題領域のみを切り出してから再実行
4. リトライで肢が生成された場合でも、実画像と肢内容を突合せて 1 肢ずつ確認するまで `reviewed_import.json` へは入れない
