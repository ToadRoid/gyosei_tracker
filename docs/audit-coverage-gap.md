# 監査カバレッジ・ギャップ分析

audit v4 の検知範囲と限界を整理する。
「現行ルールベース監査で未検出」は「誤りゼロ」を意味しない。

## 1. 検知できる誤り類型

### 1.1 critical 層

| ルール | 検知方法 | 信頼性 |
|--------|---------|--------|
| answerBoolean と explanation 結論の逆転 | 「本肢は正しい/誤り」等のフレーズとanswerBooleanの不一致 | **高** — フレーズが明示的なら確実に拾う |
| 条文明文と真逆の explanation | questionText と explanation の特定フレーズペア逆転 | **中** — 登録済みペアのみ検知。未登録の逆転パターンは取りこぼす |
| 存在しない条文番号 | 正規表現で特定パターン（行訴法11条n号等） | **低** — ハードコードされた少数パターンのみ |

### 1.2 legal_inconsistency 層

| ルール | 検知方法 | 信頼性 |
|--------|---------|--------|
| 旧法混入（出訴期間3か月） | explanation内の「3か月」+セクション名/条文番号 | **高** — 行訴法14条の出訴期間に限定 |
| 旧法混入（審査請求60日） | sectionTitle「審査請求」+ explanation「60日」 | **高** — 行審法18条に限定 |
| 旧法混入（被告適格） | sectionTitle「被告適格」+ explanation「行政庁を被告」 | **中** — 除外条件が多く、偽陰性の余地あり |
| 義務/任意の逆転 | questionText「しなければならない」vs explanation「できるにすぎない」 | **中** — 限定的なフレーズマッチのみ |
| 第三者効/拘束力の混同 | questionText「第三者」+ explanation「33条のみ」 | **中** — 行訴法32条/33条に限定 |

### 1.3 explanation_quality 層

| ルール | 検知方法 | 信頼性 |
|--------|---------|--------|
| 解説が短すぎる（<30文字） | 文字数カウント | **高** — 確実 |
| 「原本照合待ち」マーカー残存 | 文字列マッチ | **高** |
| 審査請求前置主義を原則とする記述 | フレーズマッチ | **高** |

### 1.4 ocr_suspect 層

| ルール | 検知方法 | 信頼性 |
|--------|---------|--------|
| 問題文15文字未満 | 文字数カウント | **高** |
| 助詞の異常連続（4文字以上） | 正規表現 | **中** — 特定パターンのみ |

## 2. 検知しにくい誤り類型（偽陰性リスク）

### 2.1 answer_explanation_mismatch の取りこぼし

**リスク: 高**

現行ルールは「本肢は正しい/誤り」等の明示的フレーズに依存。以下は検知できない:

- explanation が結論を明示せず、理由のみ述べているケース
  - 例: 「〜は認められない」（answerBoolean=trueだが、問題文の否定形に対応している可能性）
- explanation が婉曲表現を使うケース
  - 例: 「この点については疑問がある」（誤りを示唆しているがフレーズマッチしない）
- explanation が正しいが answerBoolean が間違っている（結論フレーズがないため検知不可）

### 2.2 旧法混入の取りこぼし

**リスク: 中〜高**

現行ルールは以下の3パターンのみカバー:
- 出訴期間3か月（行訴法14条）
- 審査請求期間60日（行審法18条）
- 被告適格の行政庁（行訴法11条）

**未カバーの旧法混入パターン:**
- 行政不服審査法の旧法全般（異議申立制度の廃止、再調査の請求の新設等）
- 行政手続法の改正（意見公募手続の追加等）
- 民法改正（債権法改正2020年施行分：消滅時効、法定利率、瑕疵担保→契約不適合等）
- 個人情報保護法の度重なる改正
- 行政書士法改正2026年1月

### 2.3 条文番号・制度名の誤り

**リスク: 高**

現行ルールは「行訴法11条のn号」のみ検知。以下は検知できない:
- 条文番号の単純な間違い（例: 14条と書くべきところ15条と記載）
- 項号の取り違え（例: 1項を2項と誤記）
- 法律名の取り違え（行訴法と行審法の混同）
- 判例の年月日の誤り

### 2.4 explanationの内容的誤り

**リスク: 最高**

これが最大の盲点。現行監査は**explanation の内容自体の法的正確性を検証しない**。

- explanation が「それっぽいが法的には誤っている」ケース
- 類似制度の要件を取り違えているケース（例: 取消訴訟と無効確認訴訟の要件混同）
- 判例の趣旨を誤解しているケース
- 正しい条文を引用しているが、適用場面が異なるケース
- 「原則」と「例外」の逆転

**これらは文字列パターンマッチでは原理的に検知不可能であり、法学知識を持つ人間（またはLLM）による確認が必要。**

### 2.5 questionText の誤り

**リスク: 中**

- OCR由来の文字化けで意味が変わっているケース（「裁」が欠落して「判所」になる等）
- 問題文自体が原本と異なるケース
- 肢の取り違え（問題文Aの解説が問題文Bについて述べている）

### 2.6 chapterCandidate / sectionTitle の誤分類

**リスク: 低〜中**

- LLM解析時に誤った章・セクションに振り分けられたケース
- 章サマリーの数字は正しいが、実態と異なる章に計上されている

## 3. 原理的に人間確認が必要な類型

| 類型 | 理由 |
|------|------|
| explanation の法的正確性 | 条文・判例・制度趣旨の正誤は法学知識が必要 |
| answerBoolean が結論フレーズなしで誤っている | パターンマッチでは検出不能 |
| 問題文と解説の「論点ずれ」 | 表面的には矛盾がないが別の論点を説明している |
| 原本との照合 | テキストデータだけでは確認不能 |
| 最新判例・法改正への対応 | ルール追加が必要（自動化不能） |

---

## 4. 監査の信頼性評価

| レイヤー | 検知率（推定） | 偽陰性リスク | 偽陽性リスク |
|---------|-------------|------------|------------|
| critical | 60-70% | **高** — フレーズなしの正誤逆転を取りこぼす | 低 |
| legal_inconsistency | 30-40% | **高** — カバーしている法改正が限定的 | 低 |
| explanation_quality | 80-90% | 低 — 文字数・マーカーは確実に拾う | 中 — 短くても正確な解説を誤検知 |
| ocr_suspect | 50-60% | **中** — 部分的な文字化けは検知しにくい | 低 |

**総合評価: 現行監査は「明確なパターン誤りの検知」に強いが、「内容の法的正確性」は検証していない。**
**「ルールベース監査で未検出」は「法的に正確」を意味しない。**

---

## 5. 改善の方向性

### 短期（ルール追加で改善可能）
- 旧法混入パターンの追加（民法改正、行審法全般）
- 条文番号検証の拡充（主要法律の条文数上限チェック）
- explanation 内の否定/肯定パターンの網羅性向上

### 中期（LLM活用）
- サンプリングした問題をLLMに投げて法的正確性を検証
- explanation の論点ずれ検出
- 新しい法改正の自動チェック

### 長期（人間レビュー体制）
- 章追加時の代表問題の原本照合
- 過去問出題年度と現行法の差分確認
- 高リスク章の定期レビューサイクル
